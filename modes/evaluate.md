# Evaluate (Sonnet) — A–F+G

Write the report as markdown, in this exact section order. No preamble, no
summary, no disclaimers. Keep each section tight — this is for the user to
read in 90 seconds.

## Header

```
# {Role} — {Company}
**Date:** YYYY-MM-DD
**Score:** X.X/5
**Archetype:** Primary / Secondary
**URL:** ...
**Legitimacy:** high | medium | low
```

## A) Role summary
Archetype, function, level, team context, remote policy, reporting line if stated.

## B) Match against CV
Bullet list: requirement → concrete evidence from CV (cite the line). Only
include requirements where match is clear or the gap is meaningful.

## C) Level & strategy
Calibrate stated level vs. user's career stage. Apply / stretch / reach?
How should the CV framing emphasize seniority without overreaching?

## D) Comp & market
If comp is stated, quote it and compare to the user's band. If not, note the
typical band for this role/market. One sentence on total comp signal
(equity, bonus) if visible.

## E) Customization plan
Top 5 CV edits + top 5 LinkedIn surfacing moves. Action-oriented.

## F) Interview prep
6–10 STAR+R stories drawn from the story bank (by tag) plus 2–3 new ones
specific to this JD's requirements. Each story: one line title + one-line
result.

## G) Posting legitimacy
Fresh? Repost? Ghost? Layoff-adjacent? Cite signals you observed.

Close with one line: **Recommendation:** apply | stretch | pass, and why in ≤10 words.
